## 🎯 How It Works

### Login Flow
```
User enters email & password
        ↓
Frontend validates input
        ↓
Sends POST /api/auth/login to backend
        ↓
Backend validates credentials
        ↓
Returns JWT token + user data
        ↓
Frontend stores in AuthContext + localStorage
        ↓
Frontend routes to role-specific dashboard
        ↓
User sees personalized dashboard
```

### Three User Roles
1. **Student** → `/student-dashboard` - Track attendance
2. **Faculty/Teacher** → `/faculty-dashboard` - Manage sessions
3. **Admin** → `/admin-dashboard` - System administration

### Protected Routes
- Unauthenticated users redirected to `/login`
- Wrong role redirected to `/unauthorized`
- Correct role allowed to dashboard
