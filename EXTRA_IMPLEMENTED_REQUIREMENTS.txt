================================================================================
 SECTION A: AUTHENTICATION & ACCOUNT MANAGEMENT
================================================================================

EFR1: OTP-Based Password Reset (Forgot Password)
  - 3-step flow: Enter email → Receive 6-digit OTP → Verify OTP → Set new password
  - OTP has 10-minute expiry, 60-second resend cooldown timer
  - Backend: /api/otp/send, /api/otp/verify, /api/otp/reset-password
  - OTP stored in otp_verification table with expiry tracking
  - HTML-styled email sent via Nodemailer (Gmail in prod, Mailtrap in test)

EFR2: Admin Pre-Approval Registration System
  - Admin must pre-approve user (email + contact) before they can register
  - approved_users table stores pre-approved entries with role, department, semester
  - Registration validates email + contact_number match against approved_users
  - After registration, approved_users record is marked is_registered = TRUE
  - Admin can add, list, filter, and delete approved users

EFR3: Student Semester Promotion System
  - Admin can promote students: 1st → 2nd → 3rd → 4th → 5th → 6th → Graduated
  - PATCH /auth/admin/students/:id/promote
  - Auto-rebuilds course_enrollment for new semester's courses after promotion
  - Graduating a student deactivates their account

EFR4: Auto-Enrollment on Registration
  - When a student registers, they are automatically enrolled in all courses
    for their department + semester via course_enrollment table
  - No manual enrollment step needed

EFR5: Login Success Modal with Auto-Redirect
  - Fullscreen animated overlay with SVG checkmark animation after login
  - "Welcome, {name}!" message with progress bar
  - 3-second auto-redirect to role-based dashboard

================================================================================
 SECTION B: PROFILE MANAGEMENT
================================================================================

EFR6: Profile Photo Upload & Editor System
  - All 3 roles (student, faculty, admin) can upload profile photos
  - Photo editor with zoom slider (0.5x–3x) and position controls
  - Supports both multipart file upload AND base64 JSON body
  - File validation: JPEG/PNG/GIF/WebP only, max 5MB
  - Photo served via GET /auth/photo/:userId with Cache-Control header
  - Photo menu: View (fullscreen modal) / Change / Remove

EFR7: Editable User Profile for All Roles
  - Students can edit: name, email, phone, department, semester, section
  - Faculty can edit: name, email, phone, department, designation
  - Admin can edit: name, email, phone, department, designation
  - All via PUT /auth/profile endpoint

================================================================================
 SECTION C: QR CODE & LOCATION FEATURES
================================================================================

EFR8: GPS-Based Location Verification for Attendance
  - Faculty generates QR with their GPS coordinates (lat/lng) and a radius
  - Student's location is captured during QR scan
  - Haversine distance formula validates student is within radius
  - Configurable radius: 5/10/25/50/100/200 meters + custom
  - SKIP_LOCATION_CHECK env variable for testing bypass
  - Handles (0,0) coordinates as "location unavailable"

EFR9: Configurable QR Code Parameters
  - Faculty can set: attendance points (1-10), duration (1-60 min + custom),
    radius (5-200m + custom) when generating QR
  - These parameters are embedded in the QR code payload

EFR10: Live QR Attendance Counter (Real-Time Polling)
  - Faculty QR generation page polls GET /qr-request/:id/attendance-count
    every 2 seconds to show live present/late count
  - Live countdown timer showing QR validity remaining time

EFR11: Three QR Scanning Methods for Students
  - Camera scan: Real-time video feed with jsQR decoding at ~15fps
  - Image upload: Student can upload a QR code image from gallery
  - Manual code entry: Text input for QR code string
  - Front/back camera toggle support

================================================================================
 SECTION D: MANUAL ATTENDANCE REQUEST SYSTEM
================================================================================

EFR12: Manual Attendance Request by Students
  - Students can request manual attendance for sessions from last 7 days
  - Must provide a reason (min 10 chars, max 500 chars)
  - POST /attendance/manual-request

EFR13: Manual Attendance Request Review by Faculty
  - Faculty can view all manual requests with status tabs (Pending/Approved/Rejected/All)
  - Search by student name or subject
  - Approve: Auto-marks attendance as 'present'
  - Reject: Optional rejection note via modal

EFR14: Manual Request History for Students
  - Students can view their own manual request history
  - Status badges: Pending (yellow), Approved (green), Rejected (red)
  - Shows session details, faculty name, submission time
